# 详细实现文档（可执行实施依据）

**项目名称**：学生课堂挂机程序（Classroom Assistant）  
**目标版本**：v3.0（Java 版）  
**适用平台**：Windows/macOS（桌面优先，Android 预留扩展点）  
**技术栈**：Java 11 + JavaFX 21 + Maven + Sherpa-ONNX JNI（预留/可替换）+ LangChain4j + SLF4J/Logback + JUnit5/Mockito

## 1. 目标与范围

### 1.1 核心目标

1. **实时录音**：点击“上课”开始录音，支持停止；录音数据写入环形缓冲区用于回溯（P0）。
2. **唤醒词配置**：用户输入姓名/多个变体（逗号分隔）并保存（P0）。
3. **唤醒词检测**：实时检测唤醒词，命中时通知（P0）。
4. **安静检测（VAD）**：连续安静超过阈值提醒，可配置/可关闭（P1）。
5. **语音识别（ASR）**：触发时回溯前 N 分钟音频转文字并展示（P0）。
6. **AI 问答**：将识别文本按固定模板生成 prompt，调用大模型并以流式方式展示（P0）。
7. **模型自动下载**：首次使用缺模型时提示下载，支持断点续传/校验（P0）。
8. **多端适配**：一套业务逻辑，多端录音实现可切换（P2）。

### 1.2 不在本期范围（明确边界）

1. 不建设服务端与远程数据库；不引入 Redis 等外部缓存（遵循“如无必要，勿增实体”）。
2. 不实现用户登录/远程上报/多用户并发（本项目为单机桌面应用）。
3. 不将录音或完整识别文本自动上传；仅在用户启用 AI 问答时向用户选择的模型服务发送 **识别文本**。

## 2. 技术选型与版本约束

### 2.1 选型总览

| 层次 | 技术 | 版本 | 说明 |
|---|---|---:|---|
| UI | JavaFX（FXML + CSS） | 21+ | 跨平台统一，UI 与逻辑分离 |
| 业务逻辑 | Java | 11+ | 并发与跨平台能力强 |
| 音频录制 | Desktop：Java Sound API | - | `javax.sound.sampled` |
| KWS/ASR/VAD | Sherpa-ONNX JNI（预留） | 最新 | 真实推理需本地模型与 JNI |
| AI | LangChain4j | 0.34.0+ | 多模型统一接入，支持流式 |
| HTTP | Java HttpClient | 11+ | 内置、支持异步 |
| 日志 | SLF4J + Logback | 2.0.9+ / 1.4.11+ | 统一日志规范 |
| 测试 | JUnit5 + Mockito | 5.10.0+ / 5.8.0+ | 单测标准组合 |
| 构建 | Maven | 3.8+ | 推荐 |

### 2.2 关键强约束（必须遵循）

1. **类型安全优先**：禁止 `Object` / `<?>` 作为返回类型（极端例外除外）。
2. **通信规范**：跨模块事件禁止魔法字符串，优先 JavaFX EventType / Listener 接口。
3. **UI 线程约束**：禁止在 UI 线程执行耗时操作；UI 更新统一 `Platform.runLater()`。
4. **模型管理**：模型路径必须 **绝对路径**；模型加载必须单例/缓存，禁止重复加载。
5. **AI 调用约定**：流式输出优先；默认超时 30 秒；最多重试 3 次，间隔 5 秒；Prompt 模板固定。
6. **日志规范**：禁止 `System.out/err`，使用 SLF4J；不得记录敏感信息（Token、完整隐私文本）。
7. **测试要求**：单测覆盖率≥80%，核心业务逻辑 100%；修改已有逻辑必须同步更新测试。

## 4. 模块划分与职责定义

### 4.1 模块一览

| 模块 | 包路径 | 输入 | 输出 | 职责 |
|---|---|---|---|---|
| UI 模块 | `com.classroomassistant.ui` | 用户点击/输入 | UI 状态展示 | 页面控制器、状态绑定、弹窗 |
| 会话编排 | `com.classroomassistant.session` | UI 命令、检测事件 | 状态迁移、触发识别/问答 | 单一编排入口，避免模块互相直连 |
| 调度与守护 | `com.classroomassistant.runtime` | 任务、心跳指标 | 重启/降级动作 | 线程池管理、健康检查、熔断 |
| 音频 | `com.classroomassistant.audio` | 设备麦克风 | PCM 帧/回溯 PCM | 采集、缓冲、回溯、资源释放 |
| 语音 | `com.classroomassistant.speech` | PCM | 唤醒/安静事件、识别文本 | KWS/VAD/ASR 抽象与实现 |
| AI | `com.classroomassistant.ai` | 识别文本 | token/回答 | Prompt 构建、LangChain4j 适配、重试/熔断 |
| 存储 | `com.classroomassistant.storage` | 配置/模型/录音请求 | 配置对象、文件路径 | Preferences/Properties、模型下载校验、录音落盘 |
| 工具与常量 | `com.classroomassistant.utils`/`constants` | - | - | 校验、路径、错误码、通知 |

### 4.2 统一接口边界（核心输入/输出）

1. **AudioRecorder**：只负责“采集 + 回溯 + 监听器”，不包含检测逻辑。
2. **Detector/Recognizer**：只负责“推理/检测”，不直接操作 UI。
3. **LLMClient**：只负责“模型调用/流式输出”，不拼装业务 prompt。
4. **ClassSessionManager**：唯一编排者，负责把事件串成完整业务链路。

## 5. 核心业务流程（文字时序图）

### 5.1 首次启动与模型下载

1. `MainApp` 启动 JavaFX，加载 `MainView.fxml`。
2. `MainController.initialize()` 创建 `ClassSessionManager` 并绑定状态属性。
3. `ClassSessionManager.initialize()` 调用 `ModelRepository.checkRequiredModels()`。
4. 若模型缺失：UI 弹窗进入下载页；用户点击开始下载：
   - `ModelDownloader.downloadAll(listener)` 使用 HttpClient 分块下载到 `cache/`。
   - 下载完成后解压/校验 MD5 与大小，原子移动到 `models/`。
5. 校验通过后，状态进入 `IDLE`，允许“上课”。

### 5.2 上课：录音与检测并行

1. 用户点击“上课”：
   - `ClassSessionManager.startClass()` 启动 `AudioRecorder.startRecording()`。
   - 启动 KWS/VAD 两个检测任务，从队列消费音频帧。
2. 录音线程每帧：
   - 写入 `CircularBuffer`。
   - 投递至 `BlockingQueue`（供 KWS/VAD 使用）。

### 5.3 触发：回溯音频 → ASR → AI 流式回答

1. KWS 命中或 VAD 超时：
   - `ClassSessionManager` 进行冷却窗口判断（防抖）。
   - 调用 `audioRecorder.getAudioBefore(lookbackSeconds)` 获取回溯 PCM。
2. `SpeechRecognizer.recognizeAsync(pcm, listener)`：
   - 成功：更新 UI“课堂内容”；生成 prompt（固定模板）。
   - 失败：记录错误并 UI 提示；回到监控。
3. `LLMClient.generateAnswerAsync(prompt, AnswerListener)`：
   - token：逐步更新 UI（打字机效果）。
   - 完成：回到监控；失败按重试/熔断策略处理。

### 5.4 异常检测与自动恢复

1. `HealthMonitor` 周期检查：
   - 录音最后帧时间、队列积压、检测线程心跳。
2. 若异常：
   - 自动重启对应任务或录音链路（优先软恢复）。
   - 多次失败后进入 ERROR 状态并提示用户。

## 6. 数据流与存储设计

### 6.1 数据流

1. PCM 帧：麦克风 → AudioRecorder → CircularBuffer / BlockingQueue
2. 检测：BlockingQueue → WakeWordDetector / SilenceDetector → 事件
3. 识别：事件 → 回溯 PCM → SpeechRecognizer → 文本
4. 问答：文本 → PromptTemplate → LLMClient → token/回答

### 6.2 存储方案（不引入数据库）

1. **用户配置**：Java Preferences  
2. **应用配置**：Java Properties（`src/main/resources/config/application.properties`）  
3. **录音文件**：文件系统 `recordings/`（WAV）  
4. **模型文件**：文件系统 `models/`（ONNX）  
5. **日志**：文件系统 `logs/`（Logback 按天滚动）  
6. **下载缓存**：文件系统 `cache/`（断点续传临时文件）  

### 6.3 Preferences 键结构（等价字段表）

| Key | 类型 | 默认值 | 校验 |
|---|---|---|---|
| `user.keywords` | String | "" | 非空、逗号分隔、长度限制 |
| `audio.lookbackSeconds` | int | 240 | 1–300 |
| `vad.enabled` | boolean | true | - |
| `vad.quietThresholdSeconds` | int | 5 | 3–30 |
| `ai.provider` | String（枚举名） | `QIANFAN` | 白名单 |
| `ai.modelName` | String | provider 默认 | 非空 |
| `ai.token.encrypted` | String（base64） | "" | 允许空（未配置） |
| `ai.timeoutSeconds` | int | 30 | 1–120 |
| `ai.maxRetryCount` | int | 3 | 0–5 |
| `recording.saveEnabled` | boolean | false | - |
| `recording.retentionDays` | int | 7 | 0–30 |

### 6.4 目录规划（运行时）

推荐使用用户数据目录（避免安装目录不可写），统一由 `AppPaths` 计算：

```
{userDataDir}/
  models/
  recordings/yyyyMMdd/*.wav
  logs/yyyy-MM-dd.log
  cache/*.part
  config/overrides.properties（可选）
```

## 7. 异常处理与容错机制

### 7.1 异常分类

1. 音频异常：权限拒绝、设备占用、采集失败、格式不匹配
2. 模型异常：缺失/损坏、加载失败、JNI 调用异常
3. 网络异常：连接/读超时、TLS 错误、HTTP 非 2xx、流式中断
4. 业务异常：配置非法、状态机非法迁移、并发冲突

### 7.2 处理策略（统一流程）

1. 捕获：在边界处捕获（IO/JNI/HTTP/线程任务），内部抛自定义异常。
2. 记录：SLF4J 结构化日志，不记录敏感 Token。
3. 反馈：UI 弹窗给出可操作建议（重试/打开设置/查看日志）。
4. 降级：AI 熔断；检测模块可单独关闭；无模型进入受限模式。
5. 恢复：录音/检测链路软重启；连续失败进入 ERROR。

### 7.3 熔断与限流

1. 触发限流：冷却窗口（例如 10 秒）避免频繁 ASR/AI。
2. AI 熔断：连续失败计数达阈值进入 OPEN，短时间内直接失败并提示用户。

## 9. 核心算法伪代码

### 9.1 触发防抖与编排

```text
if now - lastTriggerAt < triggerCooldownSeconds:
  ignore
else:
  lastTriggerAt = now
  pcm = audioRecorder.getAudioBefore(lookbackSeconds)
  recognizer.recognizeAsync(pcm):
    onResult(text):
      prompt = promptTemplate.build(text)
      llm.generateAnswerAsync(prompt):
        onToken -> append
        onComplete -> done
    onError -> UI提示并回到 MONITORING
```

### 9.2 VAD 安静计时

```text
quietMillis = 0
onEachFrame(frame, durationMillis):
  hasSpeech = vad.detect(frame, durationMillis)
  if hasSpeech:
    quietMillis = 0
  else:
    quietMillis += durationMillis
    if quietMillis >= thresholdSeconds*1000:
      fireSilenceTimeout()
      quietMillis = 0
```

### 9.3 模型断点续传

```text
existing = size(tmpFile)
request range = bytes=existing-
append to tmpFile
if md5(tmpFile) != expectedMd5: error
move tmpFile -> final
```

## 11. 性能、安全与隐私

1. 性能：
   - 录音帧固定时长，队列积压保护，避免 OOM
   - 模型单例缓存，避免重复加载
2. 安全：
   - Token 加密存储（AES-GCM，密钥不明文落盘）
   - HTTPS 请求禁止跳过证书校验
3. 隐私：
   - 默认不保存录音；开启时可配置保留天数与清理
   - 默认不上传录音；仅在启用 AI 问答时发送识别文本
