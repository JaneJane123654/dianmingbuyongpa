# 变更日志 - 多模型适配与端到端测试

**日期**: 2026-01-31

**版本**: 0.4.0

## 概述

本次迭代完成了多平台 AI 模型的完整适配、异常日志增强，以及端到端集成测试的实现。

---

## 一、DeepSeek 模型适配

### 1. LLMConfig.ModelType 扩展
- 新增 `DEEPSEEK` 枚举值
- 支持通过 `baseUrl` 自定义 API 地址（默认 `https://api.deepseek.com`）

### 2. LangChain4jClient 适配
- DeepSeek 使用 OpenAI 兼容接口，复用 `OpenAiChatModel` / `OpenAiStreamingChatModel`
- 默认模型名称：`deepseek-chat`
- 完整支持同步/流式调用

---

## 二、KIMI (Moonshot) 模型适配

### 1. LLMConfig.ModelType 扩展
- 新增 `KIMI` 枚举值
- 支持通过 `baseUrl` 自定义 API 地址（默认 `https://api.moonshot.cn/v1`）

### 2. LangChain4jClient 适配
- KIMI 使用 OpenAI 兼容接口，复用 `OpenAiChatModel` / `OpenAiStreamingChatModel`
- 默认模型名称：`moonshot-v1-8k`
- 完整支持同步/流式调用

---

## 三、百度千帆 secretKey 支持

### 1. LLMConfig 增强
- 新增 `secretKey` 字段，支持千帆双密钥认证

### 2. QianfanChatModel 配置
- `.apiKey(config.getApiKey())` + `.secretKey(config.getSecretKey())` 双密钥初始化
- 解决原先仅使用 `apiKey` 导致认证失败的问题

---

## 四、异常日志增强

### 1. 统一日志格式
- 同步调用失败：`[AI_SERVICE_FAILED] 同步调用大模型失败 ...`
- 流式调用重试：`[AI_RETRY] 流式调用 失败，第N次重试 ...`

### 2. 上下文信息
- `模型类型`：OPENAI / QIANFAN / DEEPSEEK / KIMI
- `模型名称`：实际使用的模型（如 gpt-3.5-turbo）
- `错误类型`：异常类全名
- `错误信息`：异常 message

### 3. logModelError 方法
```java
private void logModelError(String callType, int attempt, Throwable e) {
    log.warn("[AI_RETRY] {} 失败，第{}次重试，模型类型={}, 模型名称={}, 错误类型={}, 错误信息={}",
        callType, attempt, config.getModelType(), resolvedModelName,
        e.getClass().getSimpleName(), e.getMessage());
}
```

## 五、端到端集成测试

### 1. 测试类

### 2. 测试用例
