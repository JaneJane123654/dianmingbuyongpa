## 二、HealthMonitor 自动恢复功能增强

### 1. 新增 HealthState 枚举
```java
public enum HealthState {
    HEALTHY,     // 健康运行中
    RECOVERING,  // 正在恢复中
    ERROR        // 错误状态（需要人工干预）
}
```

### 2. 新增功能
- **连续失败追踪**：通过 `consecutiveFailures` 计数器追踪连续失败次数
- **最大恢复尝试**：默认最多尝试 3 次自动恢复
- **恢复回调**：`setRecoveryCallback(Runnable)` 设置恢复逻辑
- **错误回调**：`setErrorCallback(Runnable)` 设置错误处理逻辑
- **状态重置**：`resetState()` 方法用于从错误状态恢复

### 3. 改进的检查逻辑
- 超时后先尝试自动恢复
- 恢复成功时自动重置状态
- 多次恢复失败后进入 ERROR 状态

## 三、ClassSessionManager 集成

### 1. 健康监控回调注册
在 `registerListenersIfNeeded()` 中设置：
- `healthMonitor.setRecoveryCallback(this::attemptAudioRecovery)`
- `healthMonitor.setErrorCallback(this::handleHealthError)`

### 2. 新增方法
- `attemptAudioRecovery()`：尝试软恢复录音链路（停止→等待→重启）
- `handleHealthError()`：进入 ERROR 状态并通知用户

### 3. 心跳标记
在 `handleAudioFrame()` 中调用 `healthMonitor.markAudioFrameReceived()` 标记音频帧接收
