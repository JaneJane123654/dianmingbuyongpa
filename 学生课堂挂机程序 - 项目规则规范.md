## 一、项目概述

### 1.1 基本名称

| 项目信息 | 说明 |
|---------|------|
| **项目名称** | 学生课堂挂机程序（Classroom Assistant） |
| **版本号** | v3.0 (Java版) |
| **项目类型** | 桌面应用（支持Android扩展） |
| **目标用户** | 需要在课堂中及时响应老师提问的学生 |
| **核心价值** | 通过语音识别、唤醒词检测和AI智能问答，帮助学生避免因走神错过重要信息 |

## 二、项目架构

### 2.1 核心特色

#### 2.1.1 技术特色

| 特色 | 说明 | 优势 |
|-----|------|------|
| **Java全栈架构** | 使用Java实现UI层（JavaFX）和业务逻辑层 | 生态成熟，跨平台支持好 |
| **Sherpa-ONNX JNI** | 通过JNI调用Sherpa-ONNX C++库 | 性能优异，无需训练支持任意中文唤醒词 |
| **LangChain4j集成** | 使用LangChain4j统一调用多种大模型 | 模型切换便捷，支持流式输出 |
| **流式输出优先** | AI回答采用SSE协议逐步输出 | 用户体验好，减少等待焦虑 |
| **前后端分离** | JavaFX UI + 业务逻辑层 + JNI底层 | 模块解耦，易于维护和扩展 |
| **跨平台支持** | 一套代码支持Android、Windows、macOS | 降低开发和维护成本 |

#### 2.1.2 业务特色

| 特色 | 说明 |
|-----|------|
| **智能唤醒** | 实时监测是否被点名（自定义唤醒词） |
| **安静检测** | 智能区分语音和环境噪音 |
| **智能问答** | 自动记录课堂内容并生成AI回答 |
| **隐私保护** | 数据本地存储，不上传至云端 |
| **离线可用** | 核心功能（唤醒词检测、安静检测、语音识别）可离线使用 |

## 三、项目特定规范

### 3.1 核心API约定

#### 3.1.1 音频录制API

| 类名 | 方法 | 说明 | 约束 |
|-----|------|------|------|
| `AudioRecorder` | `startRecording()` | 开始录音 | 自动检查麦克风权限 |
| `AudioRecorder` | `stopRecording()` | 停止录音 | 释放资源 |
| `AudioRecorder` | `isRecording()` | 是否正在录音 | 返回boolean |
| `AudioRecorder` | `getAudioBefore(int seconds)` | 获取前N秒音频 | 参数范围：1-300秒 |
| `AudioRecorder` | `addListener(AudioListener)` | 添加监听器 | 支持多个监听器 |

**代码示例**：
```java
AudioRecorder recorder = new AudioRecorder();
recorder.addListener(new AudioListener() {
    @Override
    public void onAudioReady(byte[] data) {
        // 处理音频数据
    }

    @Override
    public void onError(String error) {
        logger.error("音频录制错误：{}", error);
    }
});

recorder.startRecording();
// ... 录音逻辑
recorder.stopRecording();
```

#### 3.1.2 唤醒词检测API

| 类名 | 方法 | 说明 | 约束 |
|-----|------|------|------|
| `WakeWordDetector` | `initialize(String modelPath, String keyword)` | 初始化检测器 | modelPath必须为绝对路径 |
| `WakeWordDetector` | `detect(float[] audioData)` | 检测唤醒词 | audioData必须为16kHz浮点数组 |
| `WakeWordDetector` | `addListener(WakeWordListener)` | 添加监听器 | 检测到唤醒词时回调 |
| `WakeWordDetector` | `setKeyword(String keyword)` | 设置唤醒词 | 支持多个变体（逗号分隔） |

**代码示例**：
```java
WakeWordDetector detector = new WakeWordDetector();
detector.initialize("/path/to/models/sherpa-onnx-kws", "张三");
detector.addListener(keyword -> {
    logger.info("检测到唤醒词：{}", keyword);
    // 触发后续处理
});

// 实时检测
float[] audioData = AudioUtils.pcmToFloat(pcmData);
boolean detected = detector.detect(audioData);
```

#### 3.1.3 安静检测API

| 类名 | 方法 | 说明 | 约束 |
|-----|------|------|------|
| `SilenceDetector` | `initialize(String modelPath)` | 初始化检测器 | modelPath必须为绝对路径 |
| `SilenceDetector` | `detect(float[] audioData, int duration)` | 检测安静 | duration单位为毫秒 |
| `SilenceDetector` | `setQuietThreshold(int seconds)` | 设置安静阈值 | 默认值：10秒 |
| `SilenceDetector` | `addListener(SilenceListener)` | 添加监听器 | 安静超时时回调 |

**代码示例**：
```java
SilenceDetector detector = new SilenceDetector();
detector.initialize("/path/to/models/sherpa-onnx-vad/silero_vad.onnx");
detector.setQuietThreshold(10); // 10秒
detector.addListener(() -> {
    logger.info("已安静超过阈值");
    // 触发通知
});

// 实时检测
float[] audioData = AudioUtils.pcmToFloat(pcmData);
boolean timeout = detector.detect(audioData, 1000); // 1秒
```

#### 3.1.4 语音识别API

| 类名 | 方法 | 说明 | 约束 |
|-----|------|------|------|
| `SpeechRecognizer` | `initialize(String modelPath)` | 初始化识别器 | modelPath必须为绝对路径 |
| `SpeechRecognizer` | `recognize(byte[] audioData)` | 识别音频 | audioData必须为16-bit PCM |
| `SpeechRecognizer` | `recognizeAsync(byte[] audioData, RecognitionListener)` | 异步识别 | 支持流式识别 |

**代码示例**：
```java
SpeechRecognizer recognizer = new SpeechRecognizer();
recognizer.initialize("/path/to/models/sherpa-onnx-asr");

// 同步识别
byte[] audioData = recorder.getAudioBefore(240); // 4分钟
String text = recognizer.recognize(audioData);
logger.info("识别结果：{}", text);

// 异步识别
recognizer.recognizeAsync(audioData, new RecognitionListener() {
    @Override
    public void onResult(String text) {
        logger.info("识别结果：{}", text);
    }

    @Override
    public void onError(String error) {
        logger.error("识别错误：{}", error);
    }
});
```

#### 3.1.5 AI问答API

| 类名 | 方法 | 说明 | 约束 |
|-----|------|------|------|
| `LLMClient` | `configure(LLMConfig config)` | 配置模型 | config包含模型类型、API Key等 |
| `LLMClient` | `generateAnswer(String prompt)` | 同步生成回答 | 返回完整回答 |
| `LLMClient` | `generateAnswerAsync(String prompt, AnswerListener)` | 异步流式生成 | 使用SSE协议逐步输出 |
| `LLMClient` | `setStreamingMode(boolean streaming)` | 设置流式模式 | 默认：true |

**代码示例**：
```java
// 配置模型
LLMConfig config = new LLMConfig(LLMConfig.ModelType.OPENAI, "gpt-3.5-turbo");
config.setApiKey("your-api-key");

LLMClient client = new LLMClient(config);

// 同步调用
```

```java
// 自定义事件
public class WakeWordDetectedEvent extends Event {
    public static final EventType<WakeWordDetectedEvent> WAKE_WORD_DETECTED =
        new EventType<>(Event.ANY, "WAKE_WORD_DETECTED");

    private final String keyword;

    public WakeWordDetectedEvent(String keyword) {
        super(WAKE_WORD_DETECTED);
        this.keyword = keyword;

    }
}

// 触发事件
fireEvent(new WakeWordDetectedEvent("张三"));

// 监听事件

});
```

**示例**：

```java
// 定义监听器接口
public interface AudioListener {
    void onAudioReady(byte[] data);
    void onError(String error);
}

// 添加监听器
public class AudioRecorder {
    private List<AudioListener> listeners = new ArrayList<>();

    public void addListener(AudioListener listener) {
        Objects.requireNonNull(listener);
        listeners.add(listener);
    }

    public void removeListener(AudioListener listener) {
        listeners.remove(listener);
    }

    private void notifyAudioReady(byte[] data) {
        for (AudioListener listener : listeners) {
            listener.onAudioReady(data);
        }
    }
}
```

## 六、避免的常见问题

### 6.1 避免重复的数据加载调用

#### 6.1.1 问题描述

**问题**：在多个地方重复加载相同的数据或模型，导致内存浪费和性能下降。

#### 6.1.2 问题示例

```java
// ❌ 错误：重复加载模型
public class Service1 {
    public void process() {
        WakeWordDetector detector = new WakeWordDetector();
        detector.initialize(modelPath, keyword); // 每次调用都重新加载
        detector.detect(audioData);
    }
}

public class Service2 {
    public void process() {
        WakeWordDetector detector = new WakeWordDetector();
        detector.initialize(modelPath, keyword); // 重复加载
        detector.detect(audioData);
    }
}
```

#### 6.1.3 解决方案

**方案1：单例模式**

```java
// ✅ 正确：使用单例模式
public class WakeWordDetectorManager {
    private static volatile WakeWordDetector instance;

    public static WakeWordDetector getInstance(String modelPath, String keyword) {
        if (instance == null) {
            synchronized (WakeWordDetectorManager.class) {
                if (instance == null) {
                    instance = new WakeWordDetector();
                    instance.initialize(modelPath, keyword);
                }
            }
        }
        return instance;
    }
}
```

**方案2：依赖注入**

```java
// ✅ 正确：使用依赖注入
@Service
public class WakeWordDetectorService {
    private final WakeWordDetector detector;

    @Autowired
    public WakeWordDetectorService(WakeWordDetector detector) {
        this.detector = detector;
        this.detector.initialize(modelPath, keyword);
    }

    public void process(float[] audioData) {
        detector.detect(audioData);
    }
}
```

**方案3：缓存机制**

```java
// ✅ 正确：使用缓存机制
public class ModelCache {
    private static final Map<String, WakeWordDetector> cache = new ConcurrentHashMap<>();

    public static WakeWordDetector getDetector(String modelPath, String keyword) {
        String key = modelPath + ":" + keyword;
        return cache.computeIfAbsent(key, k -> {
            WakeWordDetector detector = new WakeWordDetector();
            detector.initialize(modelPath, keyword);
            return detector;
        });
    }
}
```

#### 6.1.4 规范要求

1. **模型加载**：所有模型（唤醒词、VAD、ASR）必须使用单例模式或缓存机制，避免重复加载
2. **配置加载**：配置文件只加载一次，后续从内存读取
3. **资源释放**：程序退出时必须释放所有资源
4. **线程安全**：单例模式必须保证线程安全（使用双重检查锁或依赖注入）

### 6.2 其他常见问题

| 问题 | 描述 | 解决方案 |
|-----|------|---------|
| **UI线程阻塞** | 在UI线程执行耗时操作 | 使用`ExecutorService`在后台线程执行，使用`Platform.runLater()`更新UI |
| **资源泄漏** | 没有释放音频、JNI等资源 | 使用`try-with-resources`或实现`AutoCloseable`接口 |
| **异常吞没** | 捕获异常但不处理 | 记录日志并向上抛出，使用自定义异常体系 |
| **魔法常量** | 使用字符串或数字常量 | 定义枚举或常量类（如`Constants.java`） |
| **硬编码路径** | 使用硬编码的文件路径 | 使用配置文件或相对路径，支持不同环境 |
