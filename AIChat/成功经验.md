## 经验1：音频环形缓冲区覆盖写入

### 问题描述

需要在录音持续写入的情况下，保证回溯读取始终可用。

### 原因分析

固定容量缓冲区在高频写入下容易出现数据顺序混乱。

### 解决方案

使用环形缓冲区并在写入超出容量时覆盖最旧数据，读取时按写入指针计算起始位置。

### 代码示例

```java
int startIndex = writeIndex - actualLength;
if (startIndex < 0) {
    startIndex += buffer.length;
}
```

### 避免方法

确保读写都通过同一把锁保护，避免指针竞争导致的数据错乱。
